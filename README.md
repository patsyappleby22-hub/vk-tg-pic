# Telegram Image Generation Bot — Gemini / Vertex AI

Асинхронный Telegram-бот на **aiogram 3.x** для генерации изображений с помощью **Google Gemini** через **Vertex AI API**.

---

## Возможности

- **Два режима генерации**: текстовый промпт → изображение, фото + описание → редактирование
- **Две модели**: `gemini-3.1-flash-image-preview` (быстрая) и `gemini-3-pro-image-preview` (качественная)
- **Чат-модель**: `gemini-3.1-flash-lite-preview` для текстовых ответов и идей
- **Креативный режим**: пошаговое создание серии изображений с помощью AI
- **Настройки пользователя**: модель, соотношение сторон, уровень мышления, разрешение, формат отправки
- **Админ-панель**: управление API-ключами через Telegram
- **Ротация ключей**: автоматическое переключение между несколькими API-ключами при 429 ошибках
- **Апскейл**: увеличение разрешения до 4K через Pillow (Lanczos)

## Установка и запуск

### Требования

- Python 3.11+
- Telegram Bot Token от [@BotFather](https://t.me/BotFather)
- Google Cloud API Key с доступом к Vertex AI

### 1. Установка зависимостей

```bash
cd telegram-bot
pip install -r requirements.txt
```

### 2. Настройка окружения

```bash
cp .env.example .env
```

Обязательные переменные:

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather |
| `GOOGLE_CLOUD_API_KEY` | API-ключ Google Cloud |

Опциональные:

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `MAX_CONCURRENT_REQUESTS` | `3` | Макс. одновременных запросов к API |
| `LOG_LEVEL` | `INFO` | Уровень логирования |

### 3. Запуск

```bash
python -m bot.main
```

Или через скрипт:

```bash
./start.sh
```

## Настройки пользователя

Каждый пользователь может настроить через меню ⚙️:

| Настройка | Варианты |
|-----------|----------|
| **Модель** | Flash (быстрая) / Pro (качественная) |
| **Соотношение сторон** | 1:1, 16:9, 9:16, 4:3, 3:4, 3:2, 2:3, 4:5, 5:4, 21:9 |
| **Уровень мышления** | Без / Лёгкий / Средний / Глубокий (только Flash) |
| **Разрешение** | Оригинал / 1080p / 2K / 4K |
| **Формат отправки** | Фото (сжатое) / Файл (PNG оригинал) |

## Аутентификация

Бот поддерживает два способа аутентификации (в порядке приоритета):

1. **API-ключи** — добавляются через админ-панель или переменную `GOOGLE_CLOUD_API_KEY`
2. **Service Account JSON** — файлы в папке `data/service_accounts/`

Несколько ключей/аккаунтов ротируются автоматически при ошибках квоты (429).
